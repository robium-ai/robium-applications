# vla-trial Implementation Plan — Part 1 (M0–M3: the robot works)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A fine-tuned SmolVLA policy drives a simulated SO-101 arm to pick a cube and place it in a box, from camera pixels and a language instruction alone, with `make smoke` asserting ≥60% success over 10 seeded episodes.

**Architecture:** MuJoCo 3.10 + the `mujoco_menagerie` SO-101 asset, wrapped in a hand-built Gymnasium env (no SO-101 env exists anywhere). A scripted IK oracle — which can see the cube's ground-truth pose — generates 75 demonstration episodes into a LeRobotDataset v3. Those train a SmolVLA fine-tune **on HF Jobs** (never locally). Eval rolls the fine-tuned policy out in the same env and asserts a success rate. Rerun logs camera, 3D scene, joint scalars, instruction, and predicted action chunk on one timeline.

**Tech Stack:** Python 3.12 · MuJoCo 3.10 · Gymnasium · LeRobot 0.6.0 (SmolVLA) · Rerun 0.34.1 · uv · HF Jobs

**Source spec:** `docs/superpowers/specs/2026-07-13-vla-trial-design.md`

**Scope of Part 1:** M0 (spike) → M1 (env + oracle + Rerun) → M2 (dataset) → M3 (fine-tune + eval + pass bar). The presentation layer — Gradio UI, chat with direct/planner toggle, gateway endpoints, demo page, Cloud Run — is **Part 2**, written after M0's numbers land, because those numbers decide whether inference is in-process or a remote policy server.

## Global Constraints

Every task's requirements implicitly include this section. Values are copied verbatim from the spec.

- **Python `>=3.12`** — hard requirement of LeRobot 0.6.0; silently breaks 3.10/3.11 envs.
- **MuJoCo `>=3.1.3`** — the menagerie SO-101 asset's floor. Use current `3.10.x`.
- **Never depend on `gym-hil`, `gym-aloha`, or `gym-xarm`.** They pin `mujoco<3.9` (`gym-xarm`: `<3.0`) and would silently *downgrade* MuJoCo below the SO-101 asset's floor. Copy their patterns; do not import them.
- **`rerun-sdk==0.34.1`** — pinned exactly; matches robium's `rerun` skill pin and the `gradio_rerun` component's requirement (Part 2 needs it).
- **`MUJOCO_GL=cgl` on macOS** for all headless rendering. `osmesa` and `egl` are **Linux-only**. In the linux/arm64 container, use `MUJOCO_GL=egl` (or `osmesa`).
- **`mjpython`, not `python`, for MuJoCo's interactive viewer on macOS.** Headless paths are unaffected; this matters only when eyeballing the sim.
- **Training NEVER runs on macOS or MPS.** MPS is ~2 hours per 20 steps; a fine-tune is 20,000 steps. Training is HF Jobs only. MPS/CPU are for *inference* and for proving the training loop *starts*.
- **75 episodes over a ~10 cm cube-spawn region.** 50 episodes over 30 cm is a *documented outright failure* ("learned the general motion but couldn't pin down grasp locations"). Density beats count. Do not "optimize" these numbers down.
- **Use the menagerie SO-101 MJCF (`robotstudio_so101`), never the upstream `TheRobotStudio/SO-ARM100` MJCF** — upstream's own README admits its gripper joint mapping is wrong.
- All run parameters live in `src/vla_trial/config.py` and are consumed by both the Makefile and pytest, so the two can never drift (pattern lifted from `apps/manip-trial`).

```python
"""M0 spike: how fast can MuJoCo render offscreen on this machine?

Spec risk 2. On macOS the only headless backend is CGL (`MUJOCO_GL=cgl`);
osmesa and egl are Linux-only. Upstream has an open discussion titled
"Offscreen rendering with mjr_render is extremely slow", so this is measured,
not assumed.

Run: make spike-render
"""

import json
import os
import platform
import time

import mujoco

from vla_trial.config import (
    IMG_H,
    IMG_W,
    RENDER_SPIKE_JSON,
    SCENE_XML,
    SPIKE_OUTPUT_DIR,
)


def _default_backend() -> str:
    return "cgl" if platform.system() == "Darwin" else "egl"


def bench_render(n_frames: int = 1000, width: int = IMG_W, height: int = IMG_H) -> dict:
    os.environ.setdefault("MUJOCO_GL", _default_backend())

    model = mujoco.MjModel.from_xml_path(str(SCENE_XML))
    data = mujoco.MjData(model)
    mujoco.mj_forward(model, data)

    with mujoco.Renderer(model, height=height, width=width) as renderer:
        renderer.update_scene(data)          # warm-up: first render builds GL state
        renderer.render()

        start = time.perf_counter()
        for _ in range(n_frames):
            renderer.update_scene(data)
            renderer.render()
        elapsed = time.perf_counter() - start

    result = {
        "fps": n_frames / elapsed,
        "n_frames": n_frames,
        "elapsed_s": elapsed,
        "width": width,
        "height": height,
        "backend": os.environ["MUJOCO_GL"],
        "platform": platform.platform(),
    }

    SPIKE_OUTPUT_DIR.mkdir(parents=True, exist_ok=True)
    RENDER_SPIKE_JSON.write_text(json.dumps(result, indent=2))
    return result
```

`src/vla_trial/spike/bench_policy.py`:

```python
"""M0 spike: SmolVLA forward-pass latency on this machine.

Spec risk 1 — the demo's core unverified assumption. HF's "runs on a MacBook"
is a marketing claim, not a measurement, and no published benchmark exists.

Measure THREE numbers:
  make spike-policy            # MPS native + CPU native
  make spike-policy-container  # CPU inside linux/arm64 — the one that decides

The container number is decisive because Docker on macOS cannot see MPS, so the
demo container is CPU on the Mac *and* CPU on Cloud Run.

The bar is POLICY_LATENCY_CEILING_S (~1 s), not 30 Hz: action chunking means one
forward pass yields ~50 actions ≈ 1.7 s of robot motion at 30 FPS.
"""

import json
import statistics
import time

import torch

from vla_trial.config import (
    BASE_POLICY_ID,
    IMG_H,
    IMG_W,
    N_JOINTS,
    POLICY_SPIKE_JSON,
    SPIKE_OUTPUT_DIR,
    TASK,
)


def _dummy_batch(device: str) -> dict:
    """One synthetic observation, shaped like the env's real output."""
    return {
        "observation.images.wrist": torch.rand(1, 3, IMG_H, IMG_W, device=device),
        "observation.images.scene": torch.rand(1, 3, IMG_H, IMG_W, device=device),
        "observation.state": torch.rand(1, N_JOINTS, device=device),
        "task": [TASK],
    }


def bench_policy(device: str = "cpu", n_passes: int = 20) -> dict:
    from lerobot.policies.smolvla.modeling_smolvla import SmolVLAPolicy

    policy = SmolVLAPolicy.from_pretrained(BASE_POLICY_ID)
    policy.to(device)
    policy.eval()

    batch = _dummy_batch(device)

    # Warm-up: first passes pay lazy-init and kernel-compile costs.
    with torch.inference_mode():
        for _ in range(2):
            policy.reset()
            policy.select_action(batch)

    timings = []
    with torch.inference_mode():
        for _ in range(n_passes):
            policy.reset()  # force a real forward pass, not a cached chunk step
            start = time.perf_counter()
            policy.select_action(batch)
            timings.append(time.perf_counter() - start)

    result = {
        "device": device,
        "policy": BASE_POLICY_ID,
        "n_passes": n_passes,
        "mean_s": statistics.mean(timings),
        "p95_s": sorted(timings)[max(0, int(0.95 * len(timings)) - 1)],
        "min_s": min(timings),
        "max_s": max(timings),
    }

    SPIKE_OUTPUT_DIR.mkdir(parents=True, exist_ok=True)
    existing = {}
    if POLICY_SPIKE_JSON.is_file():
        existing = json.loads(POLICY_SPIKE_JSON.read_text())
    existing[device] = result
    POLICY_SPIKE_JSON.write_text(json.dumps(existing, indent=2))
    return result
```

**Note on `policy.reset()` inside the loop:** SmolVLA returns an action *chunk* and then serves subsequent `select_action` calls from that cached chunk. Without `reset()` you would be timing a dict lookup, not a forward pass — and the spike would report a wonderfully fast, entirely meaningless number.

```python
"""Language-conditioned SO-101 pick-and-place in MuJoCo.

Hand-built because no SO-101 Gymnasium env exists in any maintained package,
and the one ready-made language-conditioned MuJoCo benchmark (LIBERO) is
Linux-only. Structurally modelled on gym-hil, but deliberately NOT importing it:
gym-hil pins mujoco<3.9 and would downgrade us below the menagerie SO-101 floor
of >=3.1.3.

The policy observes pixels + joint state. The cube's pose is ground truth and is
exposed ONLY via `cube_pos` for the scripted oracle — it never enters an obs.
"""

import os
import platform

import gymnasium as gym
import mujoco
import numpy as np
from gymnasium import spaces

from vla_trial.config import (
    CONTROL_FPS,
    CUBE_SPAWN_CENTER,
    CUBE_SPAWN_HALF_EXTENT,
    IMG_H,
    IMG_W,
    MAX_EPISODE_STEPS,
    N_JOINTS,
    SCENE_XML,
    SUCCESS_XY_TOL,
    SUCCESS_Z_MAX,
    TASK,
)

# macOS headless rendering is CGL-only; osmesa/egl are Linux-only.
os.environ.setdefault("MUJOCO_GL", "cgl" if platform.system() == "Darwin" else "egl")


class SO101PickEnv(gym.Env):
    metadata = {"render_modes": ["rgb_array"], "render_fps": CONTROL_FPS}

    def __init__(self, task: str = TASK):
        self.task = task
        self.model = mujoco.MjModel.from_xml_path(str(SCENE_XML))
        self.data = mujoco.MjData(self.model)

        self._renderer = mujoco.Renderer(self.model, height=IMG_H, width=IMG_W)
        # NB: the pick target's BODY name is "box" (menagerie's), even though the
        # task string calls it "the cube". The bin is ours, added in scene_pick.xml.
        self._cube_bid = mujoco.mj_name2id(
            self.model, mujoco.mjtObj.mjOBJ_BODY, CUBE_BODY
        )
        self._bin_bid = mujoco.mj_name2id(self.model, mujoco.mjtObj.mjOBJ_BODY, BIN_BODY)
        if self._cube_bid < 0 or self._bin_bid < 0:
            raise RuntimeError(
                f"scene is missing {CUBE_BODY!r} or {BIN_BODY!r} — did you add the bin "
                "to scene_pick.xml?"
            )
        # The cube's freejoint qpos address (7 dof: 3 pos + 4 quat).
        self._cube_qadr = self.model.jnt_qposadr[self.model.body_jntadr[self._cube_bid]]

        self._steps = 0

        # Actuator ranges are the ground truth for the action space.
        lo = self.model.actuator_ctrlrange[:, 0].astype(np.float32)
        hi = self.model.actuator_ctrlrange[:, 1].astype(np.float32)
        self.action_space = spaces.Box(low=lo, high=hi, shape=(N_JOINTS,), dtype=np.float32)

        self.observation_space = spaces.Dict(
            {
                "observation.images.wrist": spaces.Box(
                    0, 255, (IMG_H, IMG_W, 3), dtype=np.uint8
                ),
                "observation.images.scene": spaces.Box(
                    0, 255, (IMG_H, IMG_W, 3), dtype=np.uint8
                ),
                "observation.state": spaces.Box(
                    -np.inf, np.inf, (N_JOINTS,), dtype=np.float32
                ),
            }
        )

    # --- ground truth: oracle only, never observed --------------------------
    @property
    def cube_pos(self) -> np.ndarray:
        return self.data.xpos[self._cube_bid].astype(np.float32)

    @property
    def bin_pos(self) -> np.ndarray:
        return self.data.xpos[self._bin_bid].astype(np.float32)

    # --- gym api ------------------------------------------------------------
    def reset(self, seed: int | None = None, options: dict | None = None):
        super().reset(seed=seed)
        mujoco.mj_resetData(self.model, self.data)

        cx, cy, cz = CUBE_SPAWN_CENTER
        dx, dy = self.np_random.uniform(
            -CUBE_SPAWN_HALF_EXTENT, CUBE_SPAWN_HALF_EXTENT, size=2
        )
        self.data.qpos[self._cube_qadr : self._cube_qadr + 3] = [cx + dx, cy + dy, cz]

        mujoco.mj_forward(self.model, self.data)
        self._steps = 0
        return self._obs(), {"task": self.task, "is_success": False}

    def step(self, action: np.ndarray):
        self.data.ctrl[:] = np.clip(
            action, self.action_space.low, self.action_space.high
        )
        n_substeps = max(1, int(round(1.0 / (CONTROL_FPS * self.model.opt.timestep))))
        for _ in range(n_substeps):
            mujoco.mj_step(self.model, self.data)

        self._steps += 1
        success = self._is_success()
        truncated = self._steps >= MAX_EPISODE_STEPS and not success
        return (
            self._obs(),
            float(success),
            bool(success),
            bool(truncated),
            {"task": self.task, "is_success": bool(success)},
        )

    def close(self):
        self._renderer.close()

    # --- internals ----------------------------------------------------------
    def _obs(self) -> dict:
        return {
            "observation.images.wrist": self._render("wrist"),
            "observation.images.scene": self._render("scene"),
            "observation.state": self.data.qpos[:N_JOINTS].astype(np.float32),
        }

    def _render(self, camera: str) -> np.ndarray:
        self._renderer.update_scene(self.data, camera=camera)
        return self._renderer.render()

    def _is_success(self) -> bool:
        cube, bin_ = self.cube_pos, self.bin_pos
        in_xy = np.linalg.norm(cube[:2] - bin_[:2]) < SUCCESS_XY_TOL
        settled = cube[2] < SUCCESS_Z_MAX
        return bool(in_xy and settled)
```

`src/vla_trial/env/ik.py`:

```python
"""Damped-least-squares IK for the SO-101's end-effector site.

Used ONLY by the scripted oracle. The policy never calls this — it maps pixels
and language straight to joint targets, which is the entire point of a VLA.
"""

import mujoco
import numpy as np

DAMPING = 1e-2


def solve_ik(
    model,
    data,
    site_name: str,
    target: np.ndarray,
    n_iters: int = 60,
    tol: float = 1e-3,
    n_arm_joints: int = 5,
) -> np.ndarray:
    """Return joint targets (n_arm_joints,) that put `site_name` at `target`.

    Iterates on a scratch copy of `data` so the live sim state is untouched.
    """
    scratch = mujoco.MjData(model)
    scratch.qpos[:] = data.qpos
    mujoco.mj_forward(model, scratch)

    sid = mujoco.mj_name2id(model, mujoco.mjtObj.mjOBJ_SITE, site_name)
    if sid < 0:
        raise ValueError(f"no site named {site_name!r} in the model")

    jacp = np.zeros((3, model.nv))
    jacr = np.zeros((3, model.nv))

    for _ in range(n_iters):
        err = target - scratch.site_xpos[sid]
        if np.linalg.norm(err) < tol:
            break

        mujoco.mj_jacSite(model, scratch, jacp, jacr, sid)
        j = jacp[:, :n_arm_joints]
        # (JᵀJ + λI)⁻¹ Jᵀ e — damped so near-singular configs don't explode.
        dq = np.linalg.solve(
            j.T @ j + DAMPING * np.eye(n_arm_joints), j.T @ err
        )
        scratch.qpos[:n_arm_joints] += dq
        mujoco.mj_forward(model, scratch)

    return scratch.qpos[:n_arm_joints].copy()
```

```python
# GRIPPER_OPEN/CLOSED are NOT known: the upstream SO-ARM100 repo documents
# 0=closed/100=open but admits the MJCF does not reflect it. Read the gripper
# actuator's ctrlrange from the model and trust THAT, not any doc:
#   lo, hi = model.actuator_ctrlrange[5]
# Then determine polarity empirically (Step 6.2) and record the answer in a
# comment here.


class OraclePolicy:
    """A waypoint state machine: approach → descend → grasp → lift → place → release."""

    def __init__(self, env):
        self.env = env
        # Read the gripper's real command range from the model — the upstream
        # docs are known-wrong about polarity. Step 6.2 decides which end is
        # "open"; swap these two lines if the arm reaches correctly but never
        # holds the cube.
        lo, hi = env.model.actuator_ctrlrange[5]
        self.open_cmd = float(hi)
        self.closed_cmd = float(lo)
        self.reset()

    def reset(self) -> None:
        self._phase = 0
        self._hold = 0

    def _waypoints(self) -> list[tuple[np.ndarray, float]]:
        cube = self.env.cube_pos
        bin_ = self.env.bin_pos
        above_cube = cube + np.array([0.0, 0.0, APPROACH_HEIGHT])
        lifted = cube + np.array([0.0, 0.0, LIFT_HEIGHT])
        above_bin = bin_ + np.array([0.0, 0.0, LIFT_HEIGHT])
        return [
            (above_cube, self.open_cmd),    # 0 approach
            (cube, self.open_cmd),          # 1 descend
            (cube, self.closed_cmd),        # 2 grasp
            (lifted, self.closed_cmd),      # 3 lift
            (above_bin, self.closed_cmd),   # 4 traverse over the bin
            (above_bin, self.open_cmd),     # 5 release — cube drops into the bin
        ]

    def act(self, obs: dict) -> np.ndarray:
        waypoints = self._waypoints()
        self._phase = min(self._phase, len(waypoints) - 1)
        target, gripper = waypoints[self._phase]

        arm = solve_ik(self.env.model, self.env.data, EE_SITE, target)

        self._hold += 1
        if self._hold >= SETTLE_STEPS:
            self._hold = 0
            self._phase += 1

        return np.concatenate([arm, [gripper]]).astype(np.float32)

    @property
    def done(self) -> bool:
        return self._phase >= len(self._waypoints())


def rollout(env, seed: int, logger=None) -> dict:
    """One oracle episode. Returns success + the frames a dataset needs."""
    obs, _ = env.reset(seed=seed)
    policy = OraclePolicy(env)

    frames: list[dict] = []
    success = False

    for step in range(MAX_EPISODE_STEPS):
        action = policy.act(obs)
        frames.append(
            {
                "observation.images.wrist": obs["observation.images.wrist"],
                "observation.images.scene": obs["observation.images.scene"],
                "observation.state": obs["observation.state"],
                "action": action,
            }
        )
        if logger is not None:
            logger.log_step(step, obs, action, task=env.task)

        obs, _reward, terminated, truncated, info = env.step(action)
        success = bool(info["is_success"])
        if terminated or truncated:
            break

    return {"success": success, "frames": frames, "n_steps": len(frames)}
```

```python
"""Rerun logging — one place that knows the archetype API.

Why Rerun and not an MJPEG stream: a video rectangle shows what happened but
discards the joint states, the language instruction, and the action chunk the
policy predicted. Rerun puts all of them on one scrubable timeline, which IS the
explanation of what the VLA did — the whole point of the demo.
"""

from pathlib import Path

import numpy as np
import rerun as rr


class RerunLogger:
    def __init__(
        self,
        app_id: str = "vla_trial",
        spawn: bool = False,
        save_path: Path | None = None,
    ):
        rr.init(app_id, spawn=spawn)
        if save_path is not None:
            save_path.parent.mkdir(parents=True, exist_ok=True)
            rr.save(str(save_path))
        self._save_path = save_path

    def log_step(
        self,
        step: int,
        obs: dict,
        action: np.ndarray,
        task: str,
        chunk: np.ndarray | None = None,
    ) -> None:
        rr.set_time_sequence("step", step)

        # What the policy actually sees.
        rr.log("camera/wrist", rr.Image(obs["observation.images.wrist"]))
        rr.log("camera/scene", rr.Image(obs["observation.images.scene"]))

        # What it was told.
        rr.log("task", rr.TextLog(task))

        # Where it is, and what it decided.
        for i, q in enumerate(obs["observation.state"]):
            rr.log(f"state/joint_{i}", rr.Scalars(float(q)))
        for i, a in enumerate(action):
            rr.log(f"action/joint_{i}", rr.Scalars(float(a)))

        # The predicted chunk, if the caller has one: the VLA's lookahead.
        if chunk is not None:
            for i in range(chunk.shape[-1]):
                rr.log(f"chunk/joint_{i}", rr.Scalars(chunk[:, i].tolist()))

    def close(self) -> None:
        rr.disconnect()
```

```python
"""Roll the oracle out N times and write a LeRobotDataset v3.

Scripted-oracle collection, not teleop: we own the MJCF, so the cube's pose is
known, and 75 episodes take minutes unattended and are regenerable after any
scene tweak. Teleop would be an hour of tedium with variable quality.
"""

from pathlib import Path

from lerobot.datasets.lerobot_dataset import LeRobotDataset

from vla_trial.config import (
    CONTROL_FPS,
    DATASET_REPO_ID,
    IMG_H,
    IMG_W,
    N_EPISODES,
    N_JOINTS,
    TASK,
)
from vla_trial.env.so101_pick import SO101PickEnv
from vla_trial.oracle.scripted_pick import rollout

FEATURES = {
    "observation.images.wrist": {
        "dtype": "video",
        "shape": (IMG_H, IMG_W, 3),
        "names": ["height", "width", "channel"],
    },
    "observation.images.scene": {
        "dtype": "video",
        "shape": (IMG_H, IMG_W, 3),
        "names": ["height", "width", "channel"],
    },
    "observation.state": {
        "dtype": "float32",
        "shape": (N_JOINTS,),
        "names": [f"joint_{i}" for i in range(N_JOINTS)],
    },
    "action": {
        "dtype": "float32",
        "shape": (N_JOINTS,),
        "names": [f"joint_{i}" for i in range(N_JOINTS)],
    },
}


def record(
    n_episodes: int = N_EPISODES,
    repo_id: str = DATASET_REPO_ID,
    push: bool = False,
) -> Path:
    dataset = LeRobotDataset.create(
        repo_id=repo_id,
        fps=CONTROL_FPS,
        features=FEATURES,
        robot_type="so101",
        use_videos=True,
    )

    env = SO101PickEnv()
    kept = 0
    seed = 0
    discarded = 0
    # DISCARD/RETRY, not "skip". The oracle succeeds ~85-92% on UNSEEN spawns
    # (Task 5 review measured 9/10 on seeds outside 0-9), so a ~10% discard rate
    # here is EXPECTED, not a bug — a discarded episode is one where the cube was
    # ejected off the pedestal on an untuned corner, and it must never enter the
    # training data (that is the poison the canary exists to keep out). We keep
    # drawing fresh seeds until N clean episodes exist.
    #
    # But guard against a real regression masquerading as bad luck: if the
    # success rate collapses, something broke (a scene edit, a detuned grasp),
    # and we must fail loudly rather than loop forever writing nothing.
    max_attempts = n_episodes * 3
    while kept < n_episodes:
        attempts = kept + discarded
        if attempts >= max_attempts:
            env.close()
            raise RuntimeError(
                f"oracle success rate collapsed: {kept} kept / {attempts} attempts "
                f"({kept / attempts:.0%}) after {max_attempts} tries. Expected ~85-92%. "
                "Something regressed — re-run `make oracle` before recording."
            )
        result = rollout(env, seed=seed)
        seed += 1
        if not result["success"]:
            discarded += 1
            print(f"seed {seed - 1}: discarded (oracle miss, expected ~10%)")
            continue

        for frame in result["frames"]:
            dataset.add_frame(frame, task=TASK)
        dataset.save_episode()
        kept += 1
        print(f"episode {kept}/{n_episodes} (seed {seed - 1}, {result['n_steps']} steps)")

    env.close()
    rate = kept / (kept + discarded)
    print(f"recorded {kept} clean episodes; discarded {discarded} "
          f"(oracle success {rate:.0%} on unseen seeds)")

    if push:
        dataset.push_to_hub()

    return dataset.root
```

**On speed:** the oracle re-runs its wrist-roll scan on every control step (Task 5's
carried-forward perf note), so one rollout is ~15 s and ~85 rollouts (75 kept +
~10 discarded) is ~20 min. That is acceptable for a one-off dataset build — do
**not** block Task 7 on optimizing the oracle. If it proves intolerable, cache the
per-phase IK solution in `OraclePolicy` (solve once per waypoint transition, reuse
until the phase advances) — but only if needed.

```python
"""Roll a SmolVLA checkpoint out in the sim and score it.

Our own loop rather than `lerobot-eval`: SO101PickEnv is not a registered
LeRobot env type, and registering it buys nothing here.

The policy sees ONLY pixels + joint state + the task string. No ground truth.
That is the whole claim being tested.
"""

import json
from pathlib import Path

import numpy as np
import torch

from vla_trial.config import (
    INFERENCE_DEVICE,
    MAX_EPISODE_STEPS,
    SMOKE_EVAL_OUTPUT_DIR,
    TASK,
)
from vla_trial.env.so101_pick import SO101PickEnv


def _to_batch(obs: dict, task: str, device: str) -> dict:
    """env obs (HWC uint8) -> policy batch (1CHW float32)."""
    batch = {}
    for key in ("observation.images.wrist", "observation.images.scene"):
        img = torch.from_numpy(obs[key]).to(device)
        batch[key] = img.permute(2, 0, 1).unsqueeze(0).float() / 255.0
    batch["observation.state"] = (
        torch.from_numpy(obs["observation.state"]).unsqueeze(0).to(device)
    )
    batch["task"] = [task]
    return batch


def evaluate(
    policy_path: str,
    n_episodes: int = 10,
    seed: int = 1000,
    task: str = TASK,
    output_dir: Path = SMOKE_EVAL_OUTPUT_DIR,
    device: str = INFERENCE_DEVICE,
    logger=None,
) -> dict:
    from lerobot.policies.smolvla.modeling_smolvla import SmolVLAPolicy

    policy = SmolVLAPolicy.from_pretrained(policy_path)
    policy.to(device)
    policy.eval()

    env = SO101PickEnv(task=task)
    successes = 0
    per_episode = []

    for i in range(n_episodes):
        obs, _ = env.reset(seed=seed + i)
        policy.reset()
        success = False

        for step in range(MAX_EPISODE_STEPS):
            with torch.inference_mode():
                action = policy.select_action(_to_batch(obs, task, device))
            action = action.squeeze(0).cpu().numpy().astype(np.float32)

            if logger is not None:
                logger.log_step(step, obs, action, task=task)

            obs, _reward, terminated, truncated, info = env.step(action)
            success = bool(info["is_success"])
            if terminated or truncated:
                break

        successes += success
        per_episode.append({"seed": seed + i, "success": success})
        print(f"episode {i + 1}/{n_episodes}: {'OK' if success else 'FAIL'}")

    env.close()

    result = {
        "policy": policy_path,
        "device": device,
        "task": task,
        "n_episodes": n_episodes,
        "successes": successes,
        "success_rate": successes / n_episodes,
        "episodes": per_episode,
    }

    output_dir.mkdir(parents=True, exist_ok=True)
    (output_dir / "eval_info.json").write_text(json.dumps(result, indent=2))
    print(f"success_rate: {result['success_rate']:.0%}")
    return result
```
