# manip-trial demo page (v1, local-only) — design

**Date:** 2026-07-15
**App:** `apps/manip-trial` (backend) + `robium-website` (page, orchestrator entry)
**Status:** approved in conversation, this document records it
**Builds on:** `2026-07-15-vla-trial-demo-page-design.md` — this is the same v1
architecture applied to manip-trial, plus the checkpoint-ladder story that spec
deferred ("watch it learn").

## 1. What ships

An interactive demo page at `/demos/manip-trial/`: the visitor picks a
checkpoint from a training ladder, hits Run, and watches the ACT policy push
the T-block in the PushT sim on an embedded Rerun timeline (env frames, agent
and block state, reward curve). A **Gallery tab** in the same UI shows the
pre-computed progression story — per-rung eval videos and metrics — so the
"watch it improve" narrative is visible before running anything live.

**v1 is local-only**, mirroring the vla-trial demo v1: it works when the
website runs on localhost (`npm run dev`) with Docker up, or in direct native
mode. The deployed site shows the card and page with an honest "cloud version
not yet supported — run it locally" notice.

### Honesty constraints (load-bearing)

- Every rung's metrics come from a real 10-episode seeded eval of that exact
  checkpoint. No number on the page is invented.
- `pc_success` will likely be 0% at every rung (PushT success requires ≥95%
  coverage, beyond ACT at this scale) — the UI states this plainly. The rising
  `avg_max_reward` curve is the story, not success theater.
- The 10k top rung comes from the earlier baseline training run (2026-07-12),
  not the ladder run; it is labeled as such in the UI.

## 2. The checkpoint ladder (new training work, ~10 min total)

- One 5k-step ACT training run on MPS (~8 min on the M2 Pro),
  `save_freq=1000`, then **prune to rungs 1k / 3k / 5k**.
- Reuse the existing 10k baseline checkpoint
  (`outputs/train/act_pusht_10k/.../010000`) as the free top rung.
- 4 rungs × ~197 MB ≈ 800 MB — stays out of git (existing `outputs/` ignore),
  baked into the demo image at build time.
- Each rung gets a 10-episode seeded eval → `eval_info.json` + rollout MP4s
  (~136 KB each).
- `outputs/demo/ladder.json` manifest — rung → steps, run provenance, metrics,
  checkpoint path, video paths. Single source read by both the Gradio gallery
  and the checkpoint radio. Generated by the eval target, not hand-written.
- New Makefile targets: `train-ladder`, `eval-ladder` (evals all rungs +
  writes the manifest).

## 3. Backend (`apps/manip-trial`) — clone of vla-trial's demo module

New code in `src/manip_trial/demo/`, bootstrapped from `vla_trial/demo/`:

- **`gateway.py`** — FastAPI implementing nav-trial's session contract
  verbatim so the website's Controls/demoClient/orchestrator reuse unchanged:
  - `POST /start?session=U` — claim (idle claims takeable, busy foreign → 503)
  - `GET /status?session=U` — nav-trial's JSON shape (`claimed`, `ready`,
    `rtf: null`, `nodes: 0`, `uptime_s`, `remaining_s`, `fleet`, `log[]`);
    foreign session → 409
  - `POST /shutdown?session=U` — foreign → 403; own → exit the process
  - CORS: exact-origin reflect for `https://robium.ai` + `http://localhost:*`,
    credentials allowed
  - Prints **`DEMO READY`** (the orchestrator's `readyLog`) once the default
    checkpoint is loaded, the env is constructed, and a warm-up render is done.
- **`ui.py`** — Gradio 6 mounted at `/ui` via `gr.mount_gradio_app`:
  - Run tab: checkpoint Radio (4 rungs, each labeled with its real
    `avg_max_reward`), Run button, status line, run lock (concurrent Runs
    queue/reject), and the **`gradio_rerun` 0.34.1** viewer streaming the
    episode as it computes.
  - Gallery tab: metrics table + per-rung eval videos, rendered from
    `ladder.json`.
- **`episode_runner.py`** — single-episode rollout logging to the Rerun
  stream per step: env RGB frame, 2D agent/block poses, action, per-step and
  cumulative/max reward. Eval uses `--eval.use_async_envs=false` semantics
  (the known forkserver gotcha) — the runner drives a single sync env
  directly.
- Device auto-select: `mps` native, `cpu` in container (ACT inference is
  light; CPU is fine — ~1 s/episode eval pace measured).
- **`docker/demo.Dockerfile`** — python:3.12-slim, deps via uv, ladder
  checkpoints + gallery assets baked at build (all local files — no HF token
  needed, unlike vla-trial). CMD runs the gateway on 8765.
- Makefile: `demo` (native/MPS), `demo-image` (build `manip-trial:latest`),
  `demo-smoke` — the live-demo skill's bar: boots, `/status` reaches ready,
  foreign session 409/403, one episode on the 10k rung completes through the
  Gradio API returning finite metrics (completion is the assertion, not task
  success), `/shutdown` exits.

## 4. Two run modes

| Mode | Start | Inference | Who owns lifecycle |
| --- | --- | --- | --- |
| **Orchestrator** (default) | page's Start button → orchestrator spawns `manip-trial:latest` | CPU | orchestrator (stop/reap/budget) |
| **Direct / native MPS** | `make demo` in `apps/manip-trial` | MPS | the user's terminal |

The page's dev-only backend dropdown selects between them;
`?host=localhost:8765` forces direct. The gateway avoids container-only
assumptions (shutdown targets its own process; paths resolve from the repo).

## 5. Website (`robium-website`)

- `demo-orchestrator/src/demos.json`: `manip-trial` entry — image
  `manip-trial:latest`, `gatewayPort` 8765, `readyLog` `DEMO READY`,
  `maxInstances` **2** (ACT-CPU is far lighter than SmolVLA),
  `sessionSeconds` 1800.
- `src/pages/demos/manip-trial.astro` + workspace island reusing the
  vla-trial page shape: topbar (dev backend dropdown), Controls pane (reused),
  single Robot pane = iframe to `http://<host>/ui` once ready. No
  Terminal/Editor/Files.
- Non-localhost origins render the "run it locally" notice instead of Start.
- `Apps.astro`: manip-trial card (real stack line + description) with
  "Try the live demo →"; `tests/smoke.sh` assertions updated with the card's
  literal strings.

## 6. Testing

- `make demo-smoke` (backend, native mode) is the app-side gate.
- Website `make smoke` keeps asserting the landing/demo pages' literal
  strings, now including the manip-trial card.
- E2E by hand for v1: orchestrator mode (container) and direct mode (MPS)
  each drive one episode on the lowest (1k) and highest (10k) rung.
- Registry: manip-trial's REGISTRY.md card gains a Live demo line in the same
  commit that lands the demo.

## 7. Explicitly out of scope (deferred)

- Cloud Run deployment; egress/service-account work.
- Training beyond the existing 10k checkpoint (a 30k/100k rung can extend the
  ladder later — the manifest design absorbs new rungs without UI changes).
- Terminal / Editor / FileTree panes.
- Any robium skill edits — learnings are logged to `learnings/` during the
  build; absorption is offered at session end per the two-hats rule.
