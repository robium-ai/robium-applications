
- [live-demo] figured out from scratch (demo orchestrator build): the in-container gateway can't manage its own lifecycle (it dies with the container it would need to restart), and a browser can't start containers. The missing piece locally is a host-level **orchestrator** — a small always-on service (Node/TS + dockerode) that owns start/stop/list/limit across demos and hands the browser a per-instance connect host; the browser then talks to that instance's gateway directly (lifecycle-only, never in the data path). This is the exact role Cloud Run's control plane plays in prod, and a clean `Driver` interface lets LocalDockerDriver (now) and a future CloudRunDriver share the API. Verified end-to-end: page Start spawns a fresh container on an ephemeral port → sim ready rtf 1.2 → Stop removes it, zero manual `make demo`. Strong candidate for a live-demo skill architecture reference (the gateway-vs-orchestrator split + the local/cloud driver seam).
