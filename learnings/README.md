# learnings/

Dated notes from building apps with the robium plugin — the raw input to robium's
skill-hardening loop. See CLAUDE.md for the capture taxonomy (frictions, trigger misses,
better methods, noise, ✓ successes) and the end-of-block retro. One file per day per
app: `YYYY-MM-DD.md` (add `-<app>` suffix if two apps run the same day). Hardening
sessions mark processed entries inline with `<!-- absorbed: YYYY-MM-DD -->`; files stay
in place.
